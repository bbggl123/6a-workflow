# 6A Workflow（Claude 接入）

本项目中，当用户请求以 `@6A` 或 `6A` 开头时，按 6A 工作流执行。

## 触发
- 用户输入以 `@6A` 或 `6A` 开头（如 `6A 帮我做一个 xxx`）。
- 或用户明确要求"用 6A 工作流 / Agentic SDLC / 带门控的 Agent 开发流程"。

## 激活
立即回复：`✅ 6A 工作流已激活，开始执行阶段1：Align 需求对齐`

## 加载协议
**必须读取本仓库根的 `6A.md`（权威协议）并严格遵循**，包括：
- 四角色职责分离（规划 / 执行 / 评估 / 守门员，职责不可混用）
- 守门员防幻觉 + 可信度标注（关键假设/设计/外部依赖/选型必须附四行说明）
- 六阶段质量门控与循环回退（门控不过则回退本阶段起点重做）
- 阶段4 Approve 须收到明确"确认"才进入执行（分级交还决策权）
- 审计环 / 反指标环 / 外部锚点体系

## 项目落地
若项目含 `project-skeleton/`，按其中的 `Agent.md` 路由与 `workflows/` 定义执行；各阶段产物写入 `docs/任务名/`。
单 Agent 会话以"角色切换"模拟多 Agent，但职责边界必须显式声明、不可混用。
