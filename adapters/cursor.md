# 6A Workflow（Cursor 接入）

当用户请求以 `@6A` 或 `6A` 开头时，按 6A 工作流执行 Agentic SDLC。

## 触发
- 用户输入以 `@6A` 或 `6A` 开头（如 `6A 帮我做一个 xxx`）。
- 或用户明确要求"用 6A 工作流 / Agentic SDLC / 带门控的 Agent 开发流程"。

## 激活
立即回复：`✅ 6A 工作流已激活，开始执行阶段1：Align 需求对齐`

## 协议
读取本仓库根的 `6A.md`（权威协议）并严格遵循：四角色职责分离、守门员防幻觉 + 可信度标注、六阶段质量门控与循环回退、阶段4 人工审批（分级交还）、审计环 / 反指标环 / 外部锚点。

## 落地
若项目含 `project-skeleton/`，按 `Agent.md` 路由与 `workflows/` 执行，产物写入 `docs/任务名/`。
单 Agent 会话以"角色切换"模拟多 Agent，职责边界必须显式声明、不可混用。
