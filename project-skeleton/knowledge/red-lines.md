# knowledge/red-lines.md — 硬性红线库（Shared State）

> 跨任务的共享记忆。新任务启动前必读；任务结束后新增的红线追加于此。
> 这是 **Static Graph 的冻结规则锚点**，不可被单次任务篡改。

## 角色红线
- 规划 Agent 不得写实现代码；不得替用户做最终决策。
- 执行 Agent 不得偏离 DESIGN/TASK；不得用 TODO 占位掩盖未实现。
- 评估 Agent 不得放宽验收标准；发现幻觉必须打回。

## 流程红线
- 任一阶段质量门控不过 → 循环回退，禁止带病前进。
- 阶段4 未收到明确"确认" → 不得进入阶段5。
- 缺可信度标注的任何对外输出 → 视为未完成。

## 安全红线
- 密钥/密码一律走 `.env`，禁止硬编码。
- 用户输入做校验与防注入；不向前端暴露内部错误。
- 日志脱敏，禁止记录密钥与个人隐私明文。

## Git 红线（多 Agent 共享分支 · 详见 `workflows/git-shield.md`）
- 执行 Agent 绝不 `git reset --hard` / `push -f` / `branch -D/-f` / `checkout -- .` / `clean -f`。
- 绝不创建 `.bundle` 或 `_v2_*` / `_verify_*` / `.git` 拷贝等垃圾文件。
- 提交后若发现提交消失 / HEAD 被外部改动 → **消失即停**，交 Lead 非破坏恢复（tag + reset --soft），禁止 Worker 自行重建历史或强推。
- 恢复动作只能由 Lead 在所有 Worker 停止后串行执行。
