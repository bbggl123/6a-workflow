# workflows/anchoring.md — 外部锚点体系（Grounding Anchors）

> 没有锚点的图会退化为互确认的闭环系统。6A 必须定义不可被 AI 篡改的外部验证源。

## 五类锚点
| 锚点类型 | 具体形式 | 控制方 | 不可篡改性来源 |
|---|---|---|---|
| 人类判断 | 阶段4 人工审批签字 | 用户本人 | 物理世界的人类意志 |
| 物理验证 | `exit 0` 测试 / 编译通过 / 实际运行 | 计算机硬件 | 物理定律 |
| 冻结规则 | 既有 coding standard / 安全合规基线 | 团队共识（写入 knowledge/red-lines.md） | 社会契约 |
| 数据现实 | 真实 DB 查询 / 真实 API 响应 / 生产日志 | 外部系统 | 真实世界状态 |
| 同行评审 | PR Code Review / 同事走查 | 其他人类 | 独立视角 |

## 使用规则
- 每个阶段至少触碰 **1 类以上** 不同锚点。
- 阶段6 最终验收须同时满足 **≥3 类** 不同锚点验证。
- 当所有验证来自同一锚点源（如只有 AI 自测通过）→ 触发 **「互确认警报」**，强制引入外部锚点。
